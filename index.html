EACL 2026 Abjad NLP Shared Task: Medical text classification in Arabic

A collaborative research challenge advancing Arabic natural language processing in the medical domain

Overview

The EACL 2026 Abjad NLP Shared Task focuses on advancing natural language processing capabilities for medical text classification in Arabic. This shared task brings together researchers and practitioners to develop and evaluate state-of-the-art models for processing Arabic medical texts.

Medical text classification in Arabic presents unique challenges due to the morphological complexity of the Arabic language, domain-specific terminology, and the scarcity of annotated medical corpora. This shared task aims to foster innovation in Arabic NLP by providing a standardized dataset and evaluation framework for medical text classification.

Shared Task Objectives

  • Develop robust models for classifying Arabic medical texts across multiple categories
  • Advance the state-of-the-art in Arabic natural language processing for specialized domains
  • Create a benchmark dataset for medical text classification in Arabic
  • Foster collaboration between NLP researchers and medical domain experts
  • Address the challenges of low-resource language processing in the medical domain

Participants will work with a curated dataset of Arabic medical texts and develop classification models that can accurately categorize medical documents. The task will evaluate systems on their ability to handle the linguistic nuances of Arabic while maintaining high accuracy in medical domain classification.

Task Description

Task Format

Participants will develop systems to classify Arabic medical texts into predefined categories. The task involves multi-class classification where each medical document must be assigned to exactly one category from the provided taxonomy.

Dataset Information

The shared task dataset consists of authentic Arabic medical texts collected from various medical sources. The dataset is divided into three splits:

  • Training Set: Annotated medical texts for model development and training
  • Development Set: Annotated texts for model validation and hyperparameter tuning
  • Test Set: Unannotated texts for final evaluation (labels withheld)

Data Format

The dataset will be provided in JSON format with the following structure: 

{
  "documents": [
    {
      "id": "doc_001",
      "text": "Arabic medical text content...",
      "category": "category_label",
      "metadata": {
        "source": "source_identifier",
        "length": 1234
      }
    }
  ]
}

Each document includes:

  • id: Unique document identifier
  • text: The Arabic medical text content
  • category: The classification label (provided for training/dev sets only)
  • metadata: Additional information about the document

Evaluation Metrics

Systems will be evaluated using multiple metrics to provide a comprehensive assessment of classification performance:

  • Macro-averaged F1 Score (Primary Metric): The arithmetic mean of F1 scores across all categories, giving equal weight to each category regardless of support
  • Micro-averaged F1 Score: Overall F1 score calculated globally across all predictions and labels
  • Accuracy: Percentage of correctly classified documents
  • Per-category Precision, Recall, and F1: Detailed performance metrics for each individual category

The official ranking will be based on the Macro-averaged F1 Score to ensure balanced performance across all categories, including minority classes.

Evaluation Criteria

Submissions will be evaluated based on the following criteria:

  • Classification Accuracy: Correctness of category assignments on the test set
  • Generalization: Model performance on unseen medical texts
  • Robustness: Consistent performance across different medical text types and categories
  • Reproducibility: Clear documentation and code that allows result replication

Participants are encouraged to submit system description papers detailing their approaches, which will be reviewed for presentation at the workshop.

Submission Guidelines

Submission Requirements

To participate in the shared task, teams must submit their predictions on the test set along with supporting documentation. All submissions must adhere to the following requirements:

1. Prediction File Format

Submit predictions in JSON format matching the following structure: 

{
  "team_name": "Your Team Name",
  "submission_id": "submission_001",
  "predictions": [
    {
      "id": "doc_001",
      "predicted_category": "category_label",
      "confidence": 0.95
    }
  ]
}

Each prediction must include:

  • id: Document identifier matching the test set
  • predicted_category: Your system's predicted category label
  • confidence: Confidence score between 0 and 1 (optional but recommended)

2. System Description

Include a brief system description (1-2 pages) covering:

  • Model architecture and approach
  • Preprocessing and feature engineering techniques
  • Training procedure and hyperparameters
  • External resources used (if any)
  • Development set results

3. Code and Reproducibility

Participants must provide:

  • Source code for the submitted system
  • README with setup and execution instructions
  • Requirements file listing all dependencies
  • Trained model weights or training scripts

Submission Instructions

  1. Register: Complete the registration form to receive access to the dataset and submission portal
  2. Download Data: Access the training, development, and test datasets through the provided links
  3. Develop System: Build and train your classification model using the training and development sets
  4. Generate Predictions: Run your system on the test set to generate predictions in the required format
  5. Submit: Upload your prediction file, system description, and code through the submission portal before the deadline
  6. Await Results: Evaluation results will be announced on the specified date

Submission Rules

  • Each team may submit up to 3 system runs for evaluation
  • Teams must designate one submission as their primary submission for official ranking
  • Use of external data and pre-trained models is allowed but must be clearly documented
  • Manual annotation or modification of test data is strictly prohibited
  • All submissions must be received by the deadline (no extensions)
  • Late submissions will not be evaluated for the official ranking
